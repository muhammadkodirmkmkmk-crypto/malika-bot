# Malika Bot — AI-консультант Baraka Consulting

Telegram-бот на Flask, использующий Claude API для живого консультационного
диалога по кредитным продуктам.

## Возможности

- Персона "Малика" — отвечает по-узбекски или по-русски в зависимости от языка клиента
- Расчёт ежемесячного платежа считается в Python (формула аннуитета), а не моделью —
  модель только озвучивает готовую цифру, чтобы исключить ошибки в математике
- Детектор "горячих лидов" (телефон в сообщении, явная готовность оформить) —
  автоматически пересылает такие сообщения администратору в Telegram
- Если клиент прямо спрашивает "ты бот?" — Малика честно и доброжелательно отвечает,
  не отрицая факта, и возвращает разговор к делу (см. system_prompt.py)

## Деплой на Railway

1. Создайте новый проект на Railway, подключите этот репозиторий
2. В Variables добавьте:
   - `TELEGRAM_BOT_TOKEN` — токен от @BotFather
   - `ANTHROPIC_API_KEY` — ключ Anthropic API
   - `ADMIN_CHAT_ID` — `514275093` (или другой ID, кому слать горячих лидов)
   - `CLAUDE_MODEL` — `claude-sonnet-4-6` (опционально)
3. Railway сам определит `Procfile` и запустит `gunicorn app:app`
4. После деплоя получите публичный URL (Settings → Networking → Generate Domain)
5. Локально или через Railway Shell выполните:
   ```
   python set_webhook.py https://<ваш-домен>.up.railway.app/webhook
   ```

## Структура

- `app.py` — Flask webhook и основная логика
- `claude_client.py` — вызов Anthropic API
- `telegram_client.py` — отправка сообщений в Telegram
- `system_prompt.py` — личность и правила Малики
- `utils.py` — расчёт аннуитета, детектор горячих лидов, грубое определение языка
- `storage.py` — история диалогов в памяти процесса (без БД)

## Важное ограничение

История диалогов хранится в памяти процесса — при перезапуске сервиса на Railway
обнуляется. Если нужна персистентность (например, чтобы возвращаться к диалогу
через день), скажите — добавим Postgres по тому же образцу, что и в боте
квалификации лидов.
